# Definition of Ready and Done

## Definition of Ready

A task is ready when:

- its goal and scope are explicit;
- inputs and dependencies are named;
- acceptance criteria include normal and failure paths;
- out-of-scope behavior is stated;
- required contract/version changes are identified;
- test strategy is clear;
- no unresolved architectural decision is hidden in the implementation task.

## Definition of Done

A task is done when:

- acceptance criteria are implemented and tested;
- security and authorization behavior is covered;
- logs/metrics are safe and useful;
- documentation and schemas are updated;
- migrations and compatibility impact are documented;
- formatting, static analysis and CI checks pass;
- the handoff report is complete.

## Incomplete work

Partial implementation remains explicitly marked incomplete. A passing happy-path demo is not sufficient for lifecycle, approval, storage, authentication or reconnect tasks.

# Android to API mapping

| Screen/action | API | Live events |
| --- | --- | --- |
| Home sessions | `GET /sessions` | `session.state_changed`, `run.completed` |
| Workspace detail | `GET /workspaces/{id}` | workspace summary events |
| Start session | `POST /sessions`, `POST /sessions/{id}/runs` | `run.started` |
| Send prompt | `POST /sessions/{id}/prompts` | `run.output`, state events |
| Interrupt | `POST /sessions/{id}/interrupt` | `run.completed` or interrupted |
| Approval detail | `GET /approvals/{id}` | `approval.requested` |
| Approval decision | `POST /approvals/{id}/decision` | `approval.resolved` |
| Timeline | `GET /sessions/{id}/events` | replay/live WebSocket |

Every write action supplies an idempotency key generated by the client and retained until the server outcome is known.

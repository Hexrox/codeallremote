# Progress reporting

## Status vocabulary

- `planned`: documented but not started;
- `in_progress`: implementation underway;
- `blocked`: external dependency or unresolved decision prevents safe progress;
- `in_review`: acceptance evidence available for review;
- `accepted`: criteria verified;
- `superseded`: replaced by a newer task/decision.

## Weekly report

Each milestone report includes completed task IDs, test evidence, changed contracts, open risks, blocked items, and the next smallest executable tasks. Percentage estimates must distinguish documentation, implementation and verified functionality.

## No silent progress

An agent must not mark a task accepted when it only produced scaffolding, a happy-path demo or unverified generated files. Partial work remains visible as partial work.

# M6 tasks — implementation governance

## M6-01 — Add agent task template

Create a reusable issue/template containing goal, inputs, scope, acceptance criteria, tests, security impact, migrations and handoff report.

## M6-02 — Add contract-change gate

CI must detect changed OpenAPI/event schemas and require a compatibility note, fixture update and version review.

## M6-03 — Add Definition-of-Done checklist

Require test evidence, documentation links, safe logs, migration status and known limitations before an implementation task is marked complete.

## M6-04 — Run first implementation readiness review

Review M1 tasks against the Ready/Done definitions. Split any task that still hides a technology choice or combines unrelated lifecycle/storage/adapter behavior.
